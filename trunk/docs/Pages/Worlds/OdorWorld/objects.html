Objects

 

Objects and Walls in the Odor World are stationary entities that act as a broadcasting center for a unique stimulus pattern, much like how a fine brie cheese disperses its odor around itself. How the signal disperses is reflective of the decay function established for that particular object.

Stimulus Dimensions

In the Odor World, there is a variable amount of Stimulus Dimensions, set by default to be 8. All objects broadcast customizable stimulus patterns during each iteration. These stimulus patterns are designed to be sensed by an agent when a neuron is coupled with it. A sensor can only receive only one dimension of the stimulus pattern at a time, so, except for sharing a signal dispersal, each stimulus value of an object is completely independent of other stimulus values. Since the total number of stimulus dimensions can be increased indefinitely, any number of descriptors can be used to signify the details of that particular object. Stimulus Values can be set in the Entity Dialogue

.

Miscellaneous Features

Edibility: Objects have the ability to become Edible. This means that if an agent comes blocked by the object at hand, it will take some amount of iterations for the agent to eat the object. The number of iterations it takes for the object to become eaten is set by the Bites to die parameter.

Resurrection: Objects also have the ability to be resurrected after it has been eaten by an agent. For each iteration after death, there will be some probability that the object will return to the Odor World. This probability is set by the Resurrection Probability parameter.
